---

## 内容

### 布局

```
┌──────────────────────────────────────────────┐
│                                              │
│  左列（60%）            右列（40%）           │
│                                              │
│  [Section Label]        ┌──────────────┐     │
│  INDEPENDENT DEV         │              │     │
│                          │  MacBook /   │     │
│  主标题                   │  iPhone      │     │

│                                              │
└──────────────────────────────────────────────┘
```

> **关键变化**: 从当前的纯居中单列布局改为**左文右图的双列布局**。这是 Apple / Linear / Stripe 式 Hero 的标准做法——让用户在视觉上立即感知到"专业产品"的质感。

---

### 文案

**Section Label**:
```
INDEPENDENT DEVELOPER
```

**主标题 (h1)**:
```
用 AI 和现代技术
为你构建真正好用的数字产品
```

**副标题 (p)**:
```
独立开发者 · 专注 AI 自动化 · Web 开发 · 小程序开发
为创作者和小企业提供从线上展示到业务自动化的完整技术服务
```

**CTA 按钮**:
```
[查看我的作品]  (btn--primary, btn--lg)
[立即联系]      (btn--secondary, btn--lg)
```

---

### 设计说明

| 决策 | 原因 |
|------|------|
| 左文右图双列布局 | 当前纯居中布局缺少视觉锚点。右侧放一张 MacBook mockup 或个人工作台照片，立即提升专业感和真实感。 |
| Section Label 使用英文全大写 | 微妙的"国际感"暗示。`INDEPENDENT DEVELOPER` 远比"独立开发者"更像 Apple 官网的标签用法。 |
| 主标题保留 `<br>` 断行 | 当前"用 AI 和现代技术<br>帮你构建真正好用的数字产品"方向正确，微调措辞更精准。 |
| 副标题用 `·` 分隔身份 | 3 秒内让用户完整认知"我是谁 + 做什么"。 |
| 双 CTA，主次分明 | Primary（查看作品 → 浏览型用户）、Secondary（立即联系 → 决策型用户），覆盖两条路径。 |
| 右图灰度处理 | 图片做 desaturate 处理或使用黑白 mockup，与整体黑白色系保持一致，不抢文字焦点。 |

---

### UI 建议

| 元素 | 规格 |
|------|------|
| Hero 整体 | `min-height: calc(100vh - var(--header-height))`，垂直居中内容 |
| 左列宽度 | 桌面端 55-60%，mobile 全宽 |
| 右列宽度 | 桌面端 40-45%，mobile 全宽（移到文字下方） |
| 右图展示 | MacBook mockup 展示一个网页截图、或 iPhone 展示小程序、或高质量个人工作台照片 |
| 右图圆角 | `--radius-xl` (16px) |
| 右图阴影 | `--shadow-md`（微弱分层感） |
| 左列文字左对齐 | 桌面端左对齐（不要居中），mobile 居中 |
| Section Label | 字号 `--text-xs`、字间距 `--tracking-wide`、颜色 `--color-accent`、全大写 |
| 主标题 | 字号 `--text-6xl` (60px)、字重 `--font-bold`、行高 `--leading-none` (1.15)、颜色 `--color-gray-900` |
| 副标题 | 字号 `--text-lg`、颜色 `--color-gray-600`、最大宽度 480px |
| CTA 按钮间距 | `--space-4` (16px) |
| 背景 | `--color-gray-50`（与页面背景统一） |

---

### 交互建议

| 交互 | 行为 |
|------|------|
| 页面加载 | Hero 文字区直接可见（不参与 `.reveal` 动画），右侧图片 fade-in 400ms |
| CTA hover | 已在 components.css 定义，无需额外处理 |
| 移动端 | 汉堡菜单 + Hero 全宽居中 + 图片在文字下方、缩小尺寸 |

---

### 动画建议

```
Hero 区域不参与 scroll-reveal。
右侧图片在页面加载时执行一次 fade-in：
  opacity: 0 → 1, duration: 600ms, easing: ease-out, delay: 200ms
```

> **原则**: Hero 是用户看到的第一屏，必须立即呈现。不做任何延迟呈现的动画。唯一可接受的动画是图片的淡入，且持续时间不超过 600ms。

---

### 为什么能提高成交率

1. **3秒内回答"他是谁"**: 左侧文字 + 右侧图片同时传递信息，双通道认知
2. **实物感降低防备**: 右侧的 mockup 或照片让"独立开发者"这个抽象概念变得具体、可信
3. **双 CTA 覆盖全部路径**: 想看作品的可以看，想直接联系的可以联系，不流失任何一个潜在客户
4. **专业感从第一眼开始**: 双列布局 = 有设计意识 = 不是随便做的模板站

### 设计说明

| 决策 | 原因 |
|------|------|
| 放在 Hero 之后的第一位 | Hero 说了"我能做什么"，TECH STACK 紧接着证明"我凭什么能做"——这是信任建立的第二步。 |
| Logo Grid 而非 Tag | Logo 是视觉化的信任信号。用户看到一个 logo grid，会潜意识觉得"这些我都听过/用过，这个人靠谱"。 |
| 灰度 + hover 彩色 | 保持整体设计的克制（灰白色系），交互时才呈现色彩，制造"发现感"。这是 Stripe 和 Vercel 官网常用的品牌展示技巧。 |

| 不写版本号和技术细节 | 这是首页，不是简历。简洁即可。 |

| Logo 图标 | 使用 SVG 内联图标，32×32px，默认 `--color-gray-400`，hover `--color-accent` |
| 名称 | 字号 `--text-xs`、颜色 `--color-gray-500`、居中 |
| 卡片 hover | 边框变 `--color-gray-300`、logo 变彩色、轻微上浮 2px |
| 整体背景 | `--color-white`（section--alt） |
| 最大宽度 | `--max-width-default` (1200px) |

---

### 交互建议

| 交互 | 行为 |
|------|------|
| 卡片 hover | border-color 从 gray-200 变 gray-300，logo 颜色从 gray-400 变各品牌色，卡片 translateY(-2px) |
| 无点击行为 | 纯展示，不需要链接 |

---

### 动画建议

```
整个 section 参与 scroll-reveal。
每个 Logo 卡片 stagger 入场：
  opacity 0→1 + translateY(12px→0)
  stagger delay: 30ms/个
  duration: 300ms
```

---

### 为什么能提高成交率

1. **"证明"而非"声称"**: 说"我是做 AI 的"不如展示"I use OpenAI + Claude + n8n + Dify"
2. **降低认知门槛**: 客户看到自己知道的技术，会说"哦他用的和我一样" → 信任 +1
3. **展示技术广度**: 14 个 logo 暗示"我能处理各种技术需求"
4. **可视化 = 记忆点**: 一个 logo grid 比一列文字列表更容易被记住

3. **部署能力 = 一条龙服务**: "Vercel/Cloudflare/Docker"告诉客户"我不只写代码，我还能帮你上线"。
4. **不夸大**: Tag 风格暗示"这些是我会用的"，而不是"这些我都是专家"——诚实感。

### 10 条 FAQ

#### FAQ 1
**Q**: `合作流程是什么样的？`

**A**:
```
简单六步：需求沟通 → 方案确认 → UI 设计 → 开发实现 → 测试优化 → 上线交付。
每一步你都能看到进展，不用担心进度不透明。
```

#### FAQ 2
**Q**: `怎么收费？有标准报价吗？`

**A**:
```
每个项目需求不同，报价也不同。了解具体需求后，我会出一个详细的报价方案。
收费方式灵活：固定价格（适合需求明确的项目）或按时计费（适合需要持续迭代的项目）。
```

#### FAQ 3
**Q**: `开发一个网站 / 小程序要多久？`

**A**:
```
简单项目（如 Landing Page）：1-2 周。
中等项目（如企业官网）：2-4 周。
复杂项目（如小程序+后台）：4-8 周。
具体时间会在方案阶段给出明确的时间线。
```

#### FAQ 4
**Q**: `交付后不满意怎么办？`

**A**:
```
开发过程中会定期同步，确保方向一致，不会出现"做完才说不满意"的情况。
交付后通常包含一定次数的免费修改。如果确实不匹配，可以协商退费——但目前为止还没发生过。
```

#### FAQ 5
**Q**: `后期能帮忙维护吗？`

**A**:
```
当然。提供长期维护服务——按次付费或按月合作都可以。
交付的代码结构清晰、有注释，你找其他人维护也没问题。不会把你锁死在我这里。
```

#### FAQ 6
**Q**: `源码会交付给我吗？`

**A**:
```
会。所有项目完成后，完整源码连同部署文档一并交付。
源码托管在 GitHub（私有仓库），你可以随时查看和下载。
```

#### FAQ 7
**Q**: `支持远程合作吗？`

**A**:
```
支持。我已经和来自不同城市的客户完成过远程项目。
沟通主要通过微信/Zoom/飞书，异步更新进度。地域不是问题。
```

#### FAQ 8
**Q**: `付款流程是什么样的？`

**A**:
```
通常是分阶段付款：启动时支付一部分，中期支付一部分，验收后支付尾款。
具体比例和节点在方案阶段确认，写入合作协议。
```

#### FAQ 9
**Q**: `你是一个人做还是团队？`

**A**:
```
我是一名独立开发者。这意味着你直接和我沟通、需求零损耗、反馈零延迟。
如果是大型项目需要团队配合，我可以协调合作的设计师和后端工程师一起参与。
```

#### FAQ 10
**Q**: `你接什么样的项目？有没有范围限制？`

**A**:
```
核心范围：AI 自动化、网站开发、小程序开发。
超出这个范围的项目——比如纯移动 App 原生开发、大型 SaaS 系统——我会直接告诉你能不能做。
能做就接，不能做就推荐其他开发者。不接做不了的项目。
```

### 设计说明

| 决策 | 原因 |
|------|------|
| 从 6 条扩展到 10 条 | 6 条不够覆盖客户常见的 10 个顾虑点。FAQ 越全面，咨询前被"劝退"的概率越低。 |
| FAQ 放在 Skills 之后、CTA 之前 | FAQ 是决策前的最后障碍清理——用户读完 FAQ 后如果顾虑解除，就会点击 CTA。 |
| 每个回答都具体、有数字、有承诺 | "1-2 周"、"分阶段付款"、"完整源码交付"——让潜在客户放心。 |
| 第 10 条设置边界 | "不接做不了的项目"——诚实信号。比起"什么都能做"的开发者，诚实的人更可信。 |

保持当前 Accordion 交互，无需修改。已在 `accordion.js` 中实现。

### 动画建议

无需额外动画。

### 为什么能提高成交率

1. **消除最后一公里顾虑**: 付款、源码、维护、不满意怎么办——最担心的 4 个问题全部覆盖。
2. **减少无效咨询**: FAQ 回答了 90% 的常见问题，用户联系你时已经是高质量线索。
3. **诚实边界**: "不接做不了的项目"让客户对"能做"的项目更有信心。
