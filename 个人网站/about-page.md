# 设计总则

---

## 关于页面的核心目标

用户点击"关于"时，心理状态是：

> "这个人看起来不错，但我需要确认他是不是靠谱的。"

关于页面的任务是：

1. **建立人格信任** — 让用户觉得"这是一个真实、有想法的人"
2. **确认专业性** — 让用户觉得"他确实懂这些"
3. **建立共鸣** — 让用户觉得"他理解我的处境"
4. **降低距离感** — 让用户觉得"我可以和他聊聊"

## 关于页面不能做的事

- ❌ 写简历式时间线（"2018年毕业于XX大学..."）
- ❌ 写鸡汤文（"我相信努力就会有回报..."）
- ❌ 堆砌技术名词（"精通 React、Vue、Angular、Svelte..."）
- ❌ 放自拍照（除非是高质量专业摄影）
- ❌ 写"关于我"作为标题（太模板化）
- ❌ 用进度条或百分比展示技能（廉价感）

## 设计基调

| 维度 | 方向 |
|------|------|
| 语气 | 自信但不傲慢、专业但不冰冷、友好但不随意 |
| 视觉 | 大量留白、单列窄栏（720px）、文字为主、克制装饰 |
| 节奏 | 每屏短小精悍，不在一个屏幕上塞太多信息 |
| 色彩 | 主要灰白、标题深黑、蓝色点缀仅在链接和强调处 |
| 排版 | 大标题 + 短段落、行高 1.75、字号 18px 正文 |

## 设计说明

| 决策 | 原因 |
|------|------|
| 不配图 | About Hero 不需要视觉锚点，文字本身就是"我"。配图反而分散注意力。Apple 关于页面的哲学：当你介绍自己时，让文字说话。 |
| 主标题以"不只是写代码"开头 | 第一句话就打破"开发者=写代码的"这个刻板印象，暗示"我提供的是解决方案，不仅仅是技术实现"。 |

| 大量留白 | Hero 下方的空白不是浪费——它制造了一种"停下来读"的氛围，而不是"快速扫过"。 |

## 动画

```
Hero 文字不参与 scroll-reveal。
页面加载时，标题执行一次极简 fade-in：
  opacity 0→1, duration: 500ms, easing: ease-out
副标题延迟 150ms 后同样 fade-in。
```

---

## 为什么能提高信任

1. **第一句话就打破刻板印象**: 用户在 3 秒内就知道"这个开发者不只是写代码的"
2. **克制 = 自信**: 不需要大喊大叫、不需要动画轰炸——极简的 Hero 暗示"我的专业能力不需要夸张来证明"
3. **向下滚动的动力**: 副标题点到为止，用户会想知道更多——自然向下滚动

## 四个理念

### 01 — 关注业务结果，而非技术本身

**正文**:
```
技术只是工具。我首先关心的是——这个功能能不能帮你多赚钱、
省时间、提升效率？选技术栈的第一标准不是"新不新"，而是"适不适合你的业务"。
```

**设计理由**: 这是最核心的差异化——大多数开发者谈论技术，我谈论业务结果。放在第一位。

---

### 02 — 做减法，不做加法

**正文**:
```
好的产品不是功能堆出来的，而是砍出来的。
我倾向于从一个最小可行版本开始，快速上线、获取反馈、持续迭代。
不会为了显得"做得多"而塞入你不需要的功能。
```

**设计理由**: 客户可能被"功能越多越好"的思维影响。这段话帮他们理解什么是真正好的开发方式，同时展示我有产品思维。

---

### 03 — 代码是写给人看的

**正文**:
```
你付钱买的不是一张写满代码的硬盘——而是一个可以被维护、
被迭代、被交接的软件资产。我的代码有清晰的注释、合理的结构、
完整的基础文档。以后换人接手也不头疼。
```

**设计理由**: 直接回应"开发完就跑"、"代码只有他能看懂"的顾虑。把代码质量转化为客户能理解的利益——"资产"而非"成本"。

---

### 04 — 能做的做，不能做的会说

**正文**:
```
我不会为了接项目夸大能力。能力范围内的，全力以赴；
超出范围的，直接告诉你，甚至帮你推荐更合适的开发者。
诚实是长期合作的基础。
```

**设计理由**: 这是建立信任最直接的方式。"能做的做，不能做的说"比任何自我推销都有说服力。客户见过太多满口答应的开发者——诚实反而是稀缺品质。

---

## 设计说明

| 决策 | 原因 |
|------|------|
| 竖排列表，带数字编号 | 4 个理念有内在逻辑顺序（业务→产品→工程→诚信），竖排让用户按顺序阅读，形成递进的信任感。 |
| 每项编号 + 加粗标题 | 编号让理念显得有体系（"这不是随便写的，是认真想过的"），标题一句话概括核心。 |
| 正文不超过 3 行 | 精简到极致。每个理念只保留最核心的一句话解释。 |
| 不配图标 | 理念是用文字建立的，图标反而稀释了文字的严肃感和说服力。 |

## 六类客户

每类客户：图标 + 客户类型 + 一句话需求描述。不展开、不列表、就是 6 张小卡片。

| 客户类型 | 图标 | 描述 |
|----------|------|------|
| 创作者 | `icon-user` | 需要专业线上形象展示作品、吸引客户 |
| 创业者 | `icon-zap` | 需要快速验证想法、搭建 MVP 和落地页 |
| 小企业 | `icon-globe` | 需要长期技术合作伙伴，而非一次性外包 |
| 自媒体团队 | `icon-smartphone` | 需要 AI 辅助内容生产、自动化运营 |
| 效率追求者 | `icon-sparkles` | 想让重复工作自动化，团队聚焦创造力 |
| 有想法的人 | `icon-message-circle` | 有一个想法需要技术落地，不知道从哪开始 |

---

## 设计说明

| 决策 | 原因 |
|------|------|
| 6 张小卡片，2×3 网格 | 不是"能服务所有人"，而是"我理解这些人的需求"。用户会不自觉地对号入座——"哦我就是创作者"。 |
| 每张卡片只有一句话 | 用客户的视角描述需求，而非我的视角描述能力。"我需要一个专业的线上形象"比"我提供网站开发服务"更有共鸣。 |
| 不写行业术语 | 用日常语言：创作者、创业者、小企业——而不是 B2B、SMB、SaaS。 |
| 卡片无按钮、无链接 | 纯展示——这里的目的不是转化，而是建立共鸣。 |
