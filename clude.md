# Project Objective

The purpose of this project is to build a professional personal brand website.

The website exists to:

- Build trust
- Showcase services
- Demonstrate real work
- Generate qualified client inquiries

Every decision should support these objectives.

---

# Working Style

Do not immediately start coding.

Instead, always follow this workflow.

## Step 1

Understand the request.

If requirements are unclear, ask questions.

Never make assumptions.

---

## Step 2

Analyze the task.

Explain:

- What needs to be built
- Why this solution is appropriate
- Which files will be affected
- Any potential risks

Keep the explanation concise.

---

## Step 3

Wait for confirmation if the task significantly changes the project.

Minor changes may be implemented directly.

---

## Step 4

Implement the solution.

Prefer clean, reusable, and maintainable code.

Avoid unnecessary complexity.

---

## Step 5

Review your own work.

Before considering the task complete, verify:

- Requirements are satisfied
- No unrelated files were modified
- Code remains readable
- Responsive layout is preserved
- Existing functionality is not broken

---

## Step 6

Provide a short summary.

Include:

- Files changed
- What was completed
- Any follow-up suggestions

---

# Communication Principles

Communicate like an experienced software engineer.

Be direct.

Be concise.

Avoid unnecessary explanations.

Explain trade-offs when necessary.

If a better solution exists, recommend it.

---

# Development Principles

Always prefer:

Simple solutions over complex ones.

Reusable components over duplicated code.

Readable code over clever code.

Business value over technical perfection.

Progress over perfection.

---

# What To Avoid

Do not:

- Over-engineer
- Add features without approval
- Rewrite unrelated code
- Introduce unnecessary dependencies
- Change project structure without reason

If something feels unnecessary, ask first.

---

# UI Expectations

The interface should feel:

- Modern
- Clean
- Premium
- Responsive
- Professional

Animations should enhance usability, not distract from it.

---

# Content Expectations

Content should focus on the client.

Avoid technical jargon.

Explain benefits before implementation details.

Help visitors understand how the service solves their problems.

---

# Final Reminder

The goal of this repository is not to create the most complicated website.

The goal is to create a website that makes potential clients trust the developer and encourages them to make contact.

Whenever there is a choice between complexity and clarity, choose clarity.
